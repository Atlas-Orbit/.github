# Welcome to Atlas & Orbit, an Interlink Company. 🛰️

Atlas-Orbit is building the next generation of open operations infrastructure, focusing on governance, coordination, evidence, and living documentation for distributed systems and product runtimes.

## Our Ecosystem

### 🌐 Core Projects

- **[Orbit](https://github.com/Atlas-Orbit/orbit)** — The living documentation and repo-context layer for the Atlas & Orbit ecosystem. Serves as the knowledge foundation, governance index, and architectural backbone for the entire constellation.

- **[OpenOps Core - Interlink](https://github.com/Atlas-Orbit/openops-core-interlink)** — The canonical interlink repository tracking executable substrate alignment across OpenOps Core 00.07, Hopper/GRACE, Security & Remediation Suite, and Tandem. Governs registry-to-harness doctrine and implementation boundaries.

- **[Control Tower MCP](https://github.com/Atlas-Orbit/control-tower-mcp)** — A governed Model Context Protocol bridge for coordinated build execution. Orchestrates work-tracker sync, GitHub Projects, Linear integration, manifests, receipts, and agent-agnostic builder coordination across the ecosystem.

- **[Demo Repository](https://github.com/Atlas-Orbit/demo-repository)** — A reference implementation showcasing best practices and platform capabilities for the GitHub ecosystem.

### 📋 Templates & Resources

- **[Repository Template](https://github.com/Atlas-Orbit/repo-template)** — A starter template for creating new repositories within the Atlas-Orbit ecosystem, enforcing standards and architectural alignment.

## What We're Building

Atlas-Orbit focuses on:

- **Governance** — Defined contracts, schemas, and lifecycle policies for product runtimes and operations
- **Interoperability** — Seamless bridges between tools (Linear, GitHub, Control Tower) and runtime systems
- **Coordination** — Agent-agnostic build orchestration and manifest-driven execution
- **Evidence** — Auditable runtime graphs, lineage tracking, and receipt-based verification
- **Living Documentation** — Always-current repo-context layers and architecture synthesis

## Architectural Principles

The ecosystem operates on these core principles:

- **Contracts first** — Schemas and validated IRs define intention and validation boundaries
- **Runtime mechanics** — Shared backbone for object graphs, events, lineage, and state projections
- **Separate powers** — Governance, runtime, operations, and synthesis layers remain distinct and composable
- **Receipts matter** — All coordinated actions produce structured, signed evidence
- **Agent-agnostic** — Control Tower remains neutral across builder assistants and work surfaces

## Key Concepts

- **Interlink** — A governed cross-project relationship defining alignment boundaries and operating law between systems
- **Manifest** — Intent-carrying document that drives synchronized execution across builders and work trackers
- **Receipt** — Structured evidence from coordinated operations, enabling audit and verification
- **Runtime Graph** — Shared event and object lineage backbone for state projections and traceability
- **Governance Kernel** — Central authority for topology, lifecycle, policy, and evidence validation
